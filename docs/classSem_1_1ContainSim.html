Detailed Description

Fire containment simulation object.

Contains all the information to make a complete simulation run, and when completed, display the fire perimeter in Cartesian coordinates.

ContainSim custom constructor.

ContainSim contains all the information to make a complete simulation run, and when completed, display the fire perimeter in Cartesian coordinates.

The following inputs are usually provided by the user:

Parameters
[in]reportSizeFire size at time of report (ac)
[in]reportRateFire spread rate at time of report (ch/h).
[in]diurnalROSa ROS array for each hour of the day - 24 values (ch/h)
[in]fireStartMinutesStartTimefire start time since midnight in minutes
[in]lwRatioFire length-to-width ratio.
[in]forcePointer to the ContainForce applied to the fire.
[in]tacticHeadAttack or RearAttack.
[in]attackDistForces build fireline this far from the fire edge (ch).

The following inputs can usually be fixed parameters:

Parameters
[in]minStepsMinimum number of simulation distance steps.
[in]maxStepsMaximum number of simulation distance steps.
[in]retryIf TRUE, if forces are overrun, the simulation is re-run starting with the next later attack time.
[in]maxFireSizeMax fire size (Acres). If fire reaches this size then the fire escapes.
[in]maxFireTimeMax fire time (Minutes). If fire burns for this long then the fire escapes.

◆ run()

void Sem::ContainSim::run ( void  )

Runs the simulation to completion.

The simulation is completed whenever:

  • the containment resources are overrun,
  • the fire is contained, or
  • all containment resources are exhausted.

◆ status()

Sem::Contain::ContainStatus Sem::ContainSim::status ( void  ) const

Access to the containment simulation status.

Returns
Attack tactic Sem::ContainStatus
  • Unreported = Fire started but not yet reported (init() not called)
  • Reported = Fire reported but not yet attacked (init() called)
  • Attacked = Fire attacked but not yet resolved
  • Contained = Fire contained by attacking forces
  • Overrun = Attacking forces are overrun
  • Exhausted = Fire escaped when all resources are exhausted
  • Overflow = Simulation max step overflow
